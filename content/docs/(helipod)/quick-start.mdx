## Deploy from GitHub/GitLab

Use this flow if your code is already in GitHub or GitLab.

### Step 1: Create Project

- Open **Dashboard**
- Click **New Project**
- Choose **GitHub** or **GitLab**

{/* IMAGE PLACEHOLDER: project provider selection */}
{/* Replace with: /docs/helipod/placeholder-git-provider.png */}

### Step 2: Choose Repository

- Search repository
- Select repository and branch
- Continue to deploy setup

{/* IMAGE PLACEHOLDER: repository list */}
{/* Replace with: /docs/helipod/placeholder-repository-select.png */}

### Step 3: Deploy

- Click **Deploy Now** for direct deploy
- Or click **Add Variables** first, then deploy

{/* IMAGE PLACEHOLDER: ready to deploy state */}
{/* Replace with: /docs/helipod/placeholder-ready-deploy.png */}

### Step 4: Verify Running Service

- Open project canvas
- Check deployment status
- Open service domain

{/* IMAGE PLACEHOLDER: running deployment on canvas */}
{/* Replace with: /docs/helipod/placeholder-running-service.png */}

## Deploy from Template

Use template deployment when you want to launch pre-configured stacks quickly.

### Step 1: Open Template Flow

- Open **Dashboard**
- Click **New Project**
- Choose **Deploy from Template**

{/* IMAGE PLACEHOLDER: template option */}
{/* Replace with: /docs/helipod/placeholder-template-option.png */}

### Step 2: Select Template

- Search template catalog
- Select desired template

{/* IMAGE PLACEHOLDER: template selection */}
{/* Replace with: /docs/helipod/placeholder-template-select.png */}

### Step 3: Configure and Deploy

- Fill required inputs (if any)
- Click **Deploy**

{/* IMAGE PLACEHOLDER: template configuration */}
{/* Replace with: /docs/helipod/placeholder-template-config.png */}

### Step 4: Validate Result

- Confirm project created
- Check services are healthy
- Access deployed app

{/* IMAGE PLACEHOLDER: template deployed on canvas */}
{/* Replace with: /docs/helipod/placeholder-template-deployed.png */}
